# Pokemon Data Schema Notes

当前为 MVP 本地 JSON 数据方案，目标是先支持中文名称识别、别名映射与属性克制查询。

## 当前 MVP 文件
- `pokemon_zh_index.json`：宝可梦基础索引，包含编号、中文名、英文名、属性
- `aliases_zh.json`：中文别名到宝可梦编号的映射
- `type_chart.json`：18 属性完整克制倍率表

## 后续数据库扩展方向
后续迁移到完整对战数据库时，建议拆分为以下实体或表：

### 1. Pokemon
- 全国图鉴编号
- 中文名 / 日文名 / 英文名
- 形态标识
- 属性组合
- 世代

### 2. Base Stats
- HP
- 攻击
- 防御
- 特攻
- 特防
- 速度
- 种族值总和

### 3. Abilities
- 常规特性
- 隐藏特性
- 特性说明
- 对战相关标签

### 4. Move Pool
- 升级技能
- 招式学习器技能
- 蛋招式
- 教学招式
- 招式属性、分类、威力、命中、PP

### 5. Battle Fields
- 常见定位
- 速度档位
- 常见道具
- 常见性格
- 常见努力值分布
- 弱点与抗性摘要
- 规则分级与可用性

## 建议迁移方式
1. MVP 阶段继续使用本地 JSON，保证可读与可维护。
2. 数据量扩大后，可迁移到 SQLite / PostgreSQL。
3. 数据加载层应保持统一接口，避免上层业务直接依赖底层存储格式。
